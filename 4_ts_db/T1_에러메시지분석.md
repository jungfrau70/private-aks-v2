이 에러 로그에서 핵심은 다음과 같습니다:

---

## ❗ 핵심 에러 메시지 요약

```
ORA-12170: Cannot connect. TCP connect timeout of 5000ms for host 10.241.185.68 port 1521.
```

이 메시지는 **TCP 수준에서의 연결 시도 자체가 타임아웃**된 것입니다.

---

## 🔍 이 에러에서 알 수 있는 사실

| 항목         | 내용                                           |
| ---------- | -------------------------------------------- |
| ❌ 연결 시도 결과 | **DB Listener까지 TCP 연결 자체가 실패함**             |
| 🧭 시도 위치   | `HikariCP` 커넥션 풀에서 JDBC 연결 시도 중              |
| 📶 대상      | `10.241.185.68:1521` (Oracle DB IP + 포트)     |
| ⏱️ 타임아웃    | 5초 (`TCP connect timeout of 5000ms`) 후 실패    |
| 🧨 원인 분류   | **네트워크 상의 문제 가능성 높음** (방화벽, NSG, 라우팅, ACL 등) |

---

## 🧪 무엇을 점검해야 하는가?

### ✅ 1. **AKS Pod → Oracle IP\:Port 접속 가능 여부 확인**

```bash
kubectl exec -it <pod-name> -- nc -zv 10.241.185.68 1521
```

* 성공하면: `succeeded`
* 실패하면: 네트워크/방화벽 문제 있음

---

### ✅ 2. **NSG / UDR / 방화벽 설정 확인**

| 항목                               | 확인 내용                                                                   |
| -------------------------------- | ----------------------------------------------------------------------- |
| **NSG (Network Security Group)** | AKS NodeSubnet, DBSubnet, ExpressRoute Gateway Subnet에 1521 포트 허용되어 있는지 |
| **UDR (User Defined Route)**     | AKS Pod IP → Oracle DB IP 라우팅이 정상적으로 설정되었는지                             |
| **방화벽 or ACL**                   | On-prem 방화벽에서 10.241.185.68:1521 로부터의 inbound 허용 여부                     |

---

### ✅ 3. **ExpressRoute 또는 VNet Peering 연결 상태**

* Peering 연결이 **Allow forwarded traffic** 되어 있는지 확인
* ExpressRoute Gateway → Oracle DB 사이에 ACL 등 보안 장비가 TCP drop 시킬 수도 있음

---

### ✅ 4. **Oracle Listener 상태**

* Oracle 서버에서 아래 명령어로 Listener 상태 확인:

```bash
lsnrctl status
```

* Listener가 `10.241.185.68` IP 및 `1521` 포트에서 리스닝 중인지 확인
* 서비스명이 정상적으로 등록되어 있는지 확인 (`services summary`)

---

## ⚠️ 유의: 이건 DB 자격증명 실패가 아님

* **TCP 연결조차 되지 않아서**, Oracle에 인증 요청도 전달되지 않았음
* 자격증명/비밀번호 문제였다면 `ORA-01017`, `ORA-28000` 같은 에러가 나왔을 것
