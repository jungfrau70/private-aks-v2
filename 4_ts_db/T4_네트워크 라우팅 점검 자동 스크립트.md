다음은 **AKS → Oracle DB 연결 실패** 시 원인 분석을 위해 네트워크 구성 점검을 자동화하는 **Bash + Azure CLI 스크립트**입니다.

---

## ✅ 🔍 네트워크 라우팅 점검 자동 스크립트 (Azure CLI 기반)

```bash

cat <<'EOF' > 1.sh
#!/bin/bash

# -------------------------
# 사용자 입력값 설정
# -------------------------
AKS_RG="your-aks-resource-group"
AKS_VNET="your-aks-vnet-name"
AKS_SUBNET="your-aks-subnet-name"

# Oracle DB IP (traceroute 대상)
ORACLE_DB_IP="10.241.185.68"

echo "▶ Step 1. UDR 연결 여부 확인 중..."
ROUTE_TABLE_ID=$(az network vnet subnet show \
  --resource-group "$AKS_RG" \
  --vnet-name "$AKS_VNET" \
  --name "$AKS_SUBNET" \
  --query "routeTable.id" -o tsv)

if [ -z "$ROUTE_TABLE_ID" ]; then
  echo "❌ [UDR 없음] 해당 Subnet은 System Route만 사용 중입니다."
else
  echo "✅ [UDR 연결됨] → $ROUTE_TABLE_ID"

  UDR_RG=$(echo "$ROUTE_TABLE_ID" | awk -F/ '{print $5}')
  UDR_NAME=$(echo "$ROUTE_TABLE_ID" | awk -F/ '{print $9}')

  echo "▶ Step 1-1. UDR 경로 목록:"
  az network route-table route list \
    --resource-group "$UDR_RG" \
    --route-table-name "$UDR_NAME" \
    --output table
fi

# -------------------------
echo ""
echo "▶ Step 2. VNet Peering 라우팅 설정 확인 중..."
az network vnet peering list \
  --resource-group "$AKS_RG" \
  --vnet-name "$AKS_VNET" \
  --query "[].{Peering:name, UseRemote:useRemoteGateways, AllowTransit:allowGatewayTransit, AllowForward:allowForwardedTraffic}" \
  --output table

# -------------------------
echo ""
echo "▶ Step 3. AKS 노드 NIC의 라우팅 테이블 확인 (수동 입력 필요)"

echo "📝 다음 명령어로 Node NIC 이름을 확인하세요:"
echo "az network nic list --query \"[].{Name:name, RG:resourceGroup}\" -o table"
read -p "Node NIC 이름을 입력하세요: " NIC_NAME
read -p "NIC 리소스 그룹을 입력하세요: " NODE_RG

echo "▶ Step 3-1. Effective Routes:"
az network nic show-effective-route-table \
  --resource-group "$NODE_RG" \
  --name "$NIC_NAME" \
  --output table

# -------------------------
echo ""
echo "▶ Step 4. Pod에서 Oracle DB로 traceroute 실행 예시"
echo "아래 명령어를 AKS Pod 내에서 실행하세요 (필요시 tools pod 사용):"
echo ""
echo "kubectl exec -it <pod-name> -- traceroute -n $ORACLE_DB_IP"
echo ""
echo "🔍 결과 해석:"
echo " - 중간 hop에서 * * * 반복 → 라우팅 누락 또는 ACL 차단 가능성"
echo " - 마지막 hop이 DB IP 도달 시 정상"

EOF

```


---

## 📦 실행 방법

1. 스크립트 저장 (예: `check_aks_oracle_routing.sh`)
2. 실행 권한 부여:

```bash
chmod +x check_aks_oracle_routing.sh
```

3. 실행:

```bash
./check_aks_oracle_routing.sh
```

4. 중간에 **AKS 노드 NIC 이름**은 수동 입력 필요
   → 또는 Node VM에서 직접 NIC 이름 추출 자동화도 추가 가능

---

## ✅ 활용 예시

이 스크립트는 다음 시나리오에 유용합니다:

* AKS에서 Oracle(DB) 또는 On-Prem 자원으로 통신이 안 되는 경우
* 네트워크 팀/보안 팀과 협업 시 사전 진단
* UDR/Peering 구성 누락 확인

---

필요 시 `NSG`, `ExpressRoute`, 또는 `NVA 라우터`까지 포함한 더 확장된 자동화도 도와드릴 수 있습니다. 원하시면 알려주세요.
