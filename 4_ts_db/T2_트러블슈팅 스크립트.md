아래는 **Oracle DB 연결 오류 (ORA-12170)** 발생 시, AKS Pod → DB 간의 네트워크 상태 및 환경 구성을 점검할 수 있는 **트러블슈팅 스크립트**입니다.
**bash 기반**이며, Pod 내부 또는 Azure CLI 환경에서 실행할 수 있습니다.

---

# 🧪 Oracle 연결 트러블슈팅 스크립트 (AKS Pod 환경 기준)

```bash
#!/bin/bash

DB_HOST="10.241.185.68"
DB_PORT="1521"
JDBC_URL="jdbc:oracle:thin:@${DB_HOST}:${DB_PORT}/XE"  # 필요 시 서비스명으로 수정
TMP_LOG="oracle_conn_diag_$(date +%Y%m%d%H%M%S).log"

echo "=== Oracle DB Connectivity Check ===" | tee $TMP_LOG
echo "Target: $DB_HOST:$DB_PORT" | tee -a $TMP_LOG

echo -e "\n[1] 🧭 Checking DNS resolution..." | tee -a $TMP_LOG
getent hosts $DB_HOST || nslookup $DB_HOST >> $TMP_LOG 2>&1

echo -e "\n[2] 📶 Checking TCP connectivity to Oracle DB..." | tee -a $TMP_LOG
nc -zv -w 5 $DB_HOST $DB_PORT >> $TMP_LOG 2>&1
if [[ $? -eq 0 ]]; then
    echo "✅ TCP port $DB_PORT is reachable." | tee -a $TMP_LOG
else
    echo "❌ Cannot reach $DB_HOST:$DB_PORT — check NSG, routing, or firewall!" | tee -a $TMP_LOG
fi

echo -e "\n[3] 🚀 Checking traceroute to DB..." | tee -a $TMP_LOG
which traceroute || (apt update && apt install -y traceroute) >> $TMP_LOG 2>&1
traceroute -n $DB_HOST >> $TMP_LOG 2>&1

echo -e "\n[4] 📤 Outbound IP check from Pod..." | tee -a $TMP_LOG
curl -s ifconfig.me | tee -a $TMP_LOG

echo -e "\n[5] 📋 Routing table inside Pod:" | tee -a $TMP_LOG
ip route show | tee -a $TMP_LOG

echo -e "\n[6] 🔍 Network interfaces:" | tee -a $TMP_LOG
ip addr show | tee -a $TMP_LOG

echo -e "\n=== Diagnostic Finished ===" | tee -a $TMP_LOG
echo "📄 Full log saved to $TMP_LOG"
```

---

## ⚙️ 사용 방법

1. AKS Pod 내부 또는 Debug용 Ubuntu Pod에서 실행

   * 필요한 경우 `busybox` 또는 `ubuntu` 이미지 기반 디버그 Pod 사용
2. 실행 전 `curl`, `nc`, `traceroute` 도구가 있어야 함

   * 없으면 `apt update && apt install curl netcat traceroute -y` 추가

## 🔍 스크립트 결과에서 확인할 것

| 섹션                           | 확인 포인트                                 |
| ---------------------------- | -------------------------------------- |
| `[1] DNS`                    | IP가 정확히 변환되는지 (FQDN 사용 시)              |
| `[2] TCP 연결`                 | `succeeded` 확인 (실패 시 NSG or FW 문제 가능성) |
| `[3] Traceroute`             | 라우팅 경로가 끊기지 않고 도달하는지                   |
| `[4] Outbound IP`            | NAT Gateway 또는 SNAT 사용 시 실제 IP 확인 가능   |
| `[5][6] Routing / Interface` | 라우팅 누락 여부, 인터페이스 상태 등 확인 가능            |
