## DB 트래픽 패킷 캡처 및 분석 절차


# Ubuntu/Debian 계열
apt update && apt install -y tcpdump traceroute
# RHEL/CentOS 계열
yum install -y tcpdump traceroute


### 1. 네트워크 패킷 캡처 (tcpdump)

- PostgreSQL(5432), Oracle(1521) 트래픽을 모두 캡처합니다.
- Pod 내부에서 실행:

```bash
kubectl exec network-test -- tcpdump -i any port 5432 or port 1521 -nn -w /tmp/trace.pcap
```
- 실시간으로 화면에 출력하며 확인하려면:
```bash
kubectl exec network-test -- tcpdump -i any port 5432 or port 1521 -nn
```
- 캡처를 일정 시간(예: 60초)만 하고 싶다면:
```bash
kubectl exec network-test -- timeout 60 tcpdump -i any port 5432 or port 1521 -nn -w /tmp/trace.pcap
```

### 2. 캡처 파일을 Host(로컬)로 복사

- Pod에서 Host(현재 디렉토리)로 trace.pcap 파일을 복사:
```bash
kubectl cp network-test:/tmp/trace.pcap ./trace.pcap
```

### 3. Wireshark로 분석

- Wireshark를 실행하고, `trace.pcap` 파일을 엽니다.
- 주요 분석 포인트:
    - RST, FIN 패킷(비정상 세션 종료)
    - TCP Retransmission, Dup ACK, Out-of-order 등 네트워크 품질 이슈
    - 세션 타임아웃, 연결 지연, 패킷 손실 등
    - 패킷 필터 예시: 
      - `tcp.port == 5432 || tcp.port == 1521`
      - `tcp.flags.reset == 1` (RST 패킷만)
- 타임라인, 스트림별 Follow TCP Stream 등으로 상세 분석

---

> 참고: Pod 내부에 tcpdump가 없으면 multitool, busybox, ubuntu 등 이미지로 배포하거나, 별도 설치 필요
> 분석 후 trace.pcap 파일은 보안상 바로 삭제 권장
