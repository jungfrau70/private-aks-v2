`24x7_aks_check.sh` 스크립트는 AKS 클러스터의 기본적인 상태를 점검하는 스크립트입니다. 이 스크립트는 다음을 수행합니다.

*   필수 도구 설치 여부 확인 (az, kubectl)
*   Azure 로그인 상태 확인
*   AKS 클러스터 상태 확인 (프로비저닝 상태, 전원 상태)
*   노드 상태 확인
*   파드 상태 확인
*   시스템 서비스 상태 확인

이 스크립트는 로그 파일에 결과를 저장합니다.

# AKS 24x7 긴급점검 체크리스트

## 사전 준비사항
- [ ] Azure CLI 설치 여부 확인
- [ ] kubectl 설치 여부 확인
- [ ] jq 설치 여부 확인
- [ ] Azure 로그인 상태 확인

## AKS 클러스터 상태 점검
- [ ] 클러스터 프로비저닝 상태 확인
- [ ] 클러스터 전원 상태 확인
- [ ] 클러스터 버전 확인

## 노드 상태 점검
- [ ] 모든 노드가 'Ready' 상태인지 확인
- [ ] 노드 리소스 사용량 확인 (CPU, 메모리)
- [ ] 노드 디스크 공간 확인

## 파드 상태 점검
- [ ] 모든 네임스페이스의 파드 상태 확인
- [ ] 비정상 파드 식별 (CrashLoopBackOff, Error, Pending 등)
- [ ] 시스템 파드 상태 확인 (kube-system 네임스페이스)

## 문제 해결 가이드
1. 노드 NotReady 상태
   - 노드 리소스 부족 확인
   - 네트워크 연결성 확인
   - 시스템 로그 확인

2. 파드 문제
   - 파드 로그 확인
   - 파드 이벤트 확인
   - 리소스 제한 확인

3. 클러스터 접근성 문제
   - Azure 인증 상태 확인
   - 네트워크 정책 확인
   - RBAC 설정 확인

## 결과 해석
- 로그 파일 위치: ./logs/24x7_aks_*.log
- 주요 확인 사항:
  * 클러스터 상태
  * 노드 상태
  * 파드 상태
  * 시스템 서비스 상태

## 에스컬레이션 기준
- 2개 이상의 노드가 NotReady 상태
- 5개 이상의 중요 파드가 비정상 상태
- 클러스터 접근 불가
- 시스템 서비스 장애

## 스크립트 설명

## 확인 방법

1.  `24x7_aks_check.env` 파일에 점검 대상 AKS 클러스터의 `RESOURCE_GROUP`과 `CLUSTER_NAME`을 설정합니다.
2.  스크립트를 실행합니다: `bash ./24x7_aks_check.sh`
3.  스크립트 실행 결과는 터미널과 `./logs/24x7_aks_*.log` 파일에 출력됩니다.
4.  로그 파일에서 각 점검 항목의 결과를 확인합니다.
    *   **필수 도구 설치 여부 확인**: `az`와 `kubectl`이 설치되어 있는지 확인합니다.
    *   **Azure 로그인 상태 확인**: Azure에 정상적으로 로그인되어 있는지 확인합니다.
    *   **AKS 클러스터 상태 확인**: 클러스터 프로비저닝 상태가 `Succeeded`이고 전원 상태가 `Running`인지 확인합니다.
    *   **노드 상태 확인**: `kubectl get nodes` 결과에서 모든 노드의 `STATUS`가 `Ready`인지 확인합니다. `NotReady` 상태의 노드가 있는지 확인합니다.
    *   **파드 상태 확인**: `kubectl get pods --all-namespaces` 결과에서 `Running` 또는 `Completed` 상태가 아닌 파드가 있는지 확인합니다.
    *   **시스템 서비스 확인**: `kube-system` 네임스페이스의 파드들이 정상적으로 실행 중인지 확인합니다.
