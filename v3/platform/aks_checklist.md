## 스크립트 설명

`aks_check.sh` 스크립트는 AKS 클러스터의 플랫폼 수준 상세 점검을 수행하는 스크립트입니다. 이 스크립트는 다음을 수행합니다.

*   클러스터 구성 확인 (쿠버네티스 버전, 네트워크 프로필, 애드온)
*   노드풀 확인 (자동 스케일링 설정, 최소/최대 노드 수)
*   네트워크 연결성 확인 (DNS 해석, 네트워크 정책)
*   스토리지 구성 확인 (스토리지 클래스, PVC)
*   클러스터 메트릭 확인 (노드/파드 리소스 사용량)
*   보안 설정 확인 (RBAC, Pod Security Policy)
*   로깅 및 모니터링 확인 (진단 설정, Prometheus 메트릭)

이 스크립트는 로그 파일에 결과를 저장합니다.

## 확인 방법

1.  `aks_check.env` 파일에 점검 대상 AKS 클러스터의 `RESOURCE_GROUP`과 `CLUSTER_NAME`을 설정합니다.
2.  스크립트를 실행합니다: `bash ./aks_check.sh`
3.  스크립트 실행 결과는 터미널과 `./logs/aks_platform_*.log` 파일에 출력됩니다.
4.  로그 파일에서 각 점검 항목의 결과를 확인합니다.
    *   **클러스터 구성 확인**: 쿠버네티스 버전, 네트워크 플러그인, 네트워크 정책, 모니터링 활성화 상태를 확인합니다.
    *   **노드풀 확인**: 각 노드풀의 자동 스케일링 활성화 여부, 최소/최대 노드 수를 확인합니다.
    *   **네트워크 연결성 확인**: `dns-test` 파드를 실행하여 DNS 해석이 정상적으로 작동하는지 확인합니다. 네트워크 정책 목록을 확인합니다.
    *   **스토리지 구성 확인**: 스토리지 클래스 목록과 모든 네임스페이스의 PVC 목록을 확인합니다.
    *   **클러스터 메트릭 확인**: `kubectl top nodes`와 `kubectl top pods` 명령 결과를 통해 노드 및 파드의 리소스 사용량을 확인합니다.
    *   **보안 설정 확인**: 클러스터 역할(ClusterRoles), 클러스터 역할 바인딩(ClusterRoleBindings), Pod Security Policy(PSP) 목록을 확인합니다.
    *   **로깅 및 모니터링 확인**: Azure Monitor 진단 설정이 구성되어 있는지 확인하고, ServiceMonitor 목록을 확인합니다.
